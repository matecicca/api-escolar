📖 Documentación de la API Escolar

Esta API gestiona Usuarios, Clases e Inscripciones.
Todas las respuestas son en JSON y se recomienda enviar datos con Content-Type: application/json.

👥 Usuarios (/usuarios)

GET /usuarios → Listar todos los usuarios (con filtros)
GET /usuarios/:id → Obtener usuario por ID
POST /usuarios → Crear un nuevo usuario
PUT /usuarios/:id → Actualizar usuario existente
DELETE /usuarios/:id → Eliminar usuario

Parámetros de filtrado

  • ?tipo=alumno → Solo alumnos
  • ?tipo=docente → Solo docentes
  • ?nombre=<texto> → Busca por nombre (parcial, insensible a mayúsculas, soporta espacios)
  • Combinable: ?tipo=alumno&nombre=sofia

Validaciones

  • tipo debe ser alumno o docente
  • Emails deben ser únicos

📚 Clases (/clases)

GET /clases → Listar clases (con filtros)
GET /clases/:id → Obtener clase por ID
POST /clases → Crear clase
PUT /clases/:id → Actualizar clase
DELETE /clases/:id → Eliminar clase

Parámetros de filtrado

  • ?docente=<id> → Filtra por ID de docente
  • ?docente=<email> → Filtra por email de docente
  • ?docente=<nombre> → Filtra por nombre de docente
  • ?nombre=<texto> → Busca por nombre de clase (parcial, insensible a mayúsculas)
  • Combinable: ?docente=mariana&nombre=mate

Validaciones

  • docente debe ser un usuario existente con tipo=docente
  • classCode debe ser único y numérico
  • No se pueden crear más de 15 clases

📝 Inscripciones (/inscripciones)

GET /inscripciones → Listar inscripciones (con filtros)
POST /inscripciones → Crear inscripción
DELETE /inscripciones/:id → Eliminar inscripción

Parámetros de filtrado

  • ?alumno=<id> → Filtra por ID de alumno
  • ?alumno=<email> → Filtra por email de alumno
  • ?alumno=<nombre> → Filtra por nombre de alumno
  • ?clase=<id> → Filtra por ID de clase
  • ?clase=<classCode> → Filtra por código de clase
  • ?clase=<nombre> → Filtra por nombre de clase
  • ?docente=<nombre> → Filtra por nombre del docente asociado
  • Combinable: ?alumno=Carla&clase=Historia

Validaciones

  • alumno debe ser un usuario existente con tipo=alumno
  • clase debe ser una clase existente
  • No se permiten inscripciones duplicadas (un alumno en la misma clase)

📌 Reglas Generales

  • Usar URL encoding para nombres con espacios (Sofía%20Martínez)
  • Respuestas en JSON con mensajes de error claros (400, 404, 500)
  • Filtros combinables para refinar resultados
